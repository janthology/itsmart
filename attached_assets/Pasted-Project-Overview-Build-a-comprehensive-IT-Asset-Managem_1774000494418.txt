Project Overview
Build a comprehensive IT Asset Management (ITAM) system using Next.js 16, Supabase (PostgreSQL + Auth), and shadcn/ui components. The system supports three user roles (General User, Support Staff, Administrator) with role-based access control at both the application and database (RLS) levels.
Tech Stack

Frontend: Next.js 16 with React 19 (App Router)
UI Framework: shadcn/ui components (already available) + Tailwind CSS
Database: Supabase PostgreSQL with Row-Level Security (RLS)
Authentication: Supabase Auth
Storage: Supabase Storage (for asset photos and attachments)
Client Library: @supabase/supabase-js

Database Schema Setup
Tables to Create (via migration script):


profiles (extends auth.users)

id (UUID, FK to auth.users)
full_name, role (enum), department
created_at, updated_at



assets

id, asset_tag (unique), name, category (enum), status (enum)
assigned_to (FK to profiles), purchase_date, notes
created_at, updated_at



tickets

id, title, description, status (enum), priority (enum)
asset_id (FK to assets), created_by (FK to profiles), assigned_to (FK to profiles)
created_at, updated_at, closed_at



ticket_comments

id, ticket_id (FK), created_by (FK), comment_text
created_at



categories (dropdown management)

id, name, type (asset_category or ticket_priority)
created_at



RLS Policies:

General Users: Can view only assigned assets, create tickets, see/comment on own tickets
Support Staff: Can view all assets, manage all tickets (claim, update status, comment)
Administrators: Full access to all data, manage users and system settings

Key Implementation Details
Authentication Flow:

Supabase Auth (email/password + OAuth optional)
Session stored in httpOnly cookie
Middleware checks auth on every request
Profile enrichment (role, department) from profiles table

RBAC Implementation:

Custom hooks and guards check user.role
Database RLS enforces server-side security
Admin-only pages have protective wrapper
Row filters applied at component level

Data Fetching:

Server Components for initial data load
React hooks + Supabase client for real-time updates
SWR or custom hooks for client-side state
Optimistic updates for better UX

UI/UX:

Consistent use of shadcn/ui components
Dark/light theme support via next-themes
Responsive layout with sidebar navigation
Form validation feedback with react-hook-form + zod
Toast notifications for actions
